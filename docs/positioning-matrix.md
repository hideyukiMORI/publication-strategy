# Positioning Matrix

Last updated: 2026-05-22

## Portfolio Goal

Increase **GitHub recognition** for the maintainer's OSS portfolio. Products are weapons toward that goal; priority may shift if one shows clearly stronger traction.

## Weapons

| | NeNe | NENE2 | nene2-python |
|---|---|---|---|
| **One-line** | Tiny renovated legacy PHP for reviewable small services | Modern PHP 8.4 JSON API framework | NENE2 philosophy in Python (FastAPI) |
| **Best audience** | Developers comfortable with old URL-segment MVC | Teams shipping JSON APIs with OpenAPI/MCP | Python backend developers |
| **GitHub hook** | Renovation story, demo at nene-php.com | Packagist, OpenAPI, howto library, field trials | Strict typing, MCP, large FT coverage |
| **Install path** | `git clone` (small-service base) | `git clone` or `composer require` | `pip` / `uv add` |
| **Current release** | v0.2.0 | v1.x | v1.8.x |
| **Publication state** | Zenn article done (#177); Qiita pending (#178) | No dedicated article plan in this repo yet | No dedicated article plan in this repo yet |

## Hypothesis (2026-05-22)

Primary weapon for GitHub traffic in the near term: **NENE2** — broader entry points (Packagist, MCP, howto, external field trials).

Secondary: **NeNe** — strong narrative (Zenn), narrower audience.

Tertiary for now: **nene2-python** — differentiate when targeting Python communities explicitly.

**Review date:** 2026-08-22 (revisit after Qiita + first metrics from NeNe Zenn).

## Do Not Mix (initial rule)

- One article = one product story until baseline metrics exist.
- Cross-links in profile README and article footers are OK; combined "three frameworks" launch posts are not.

## Success Signals (GitHub-focused)

Track weekly in experiment logs:

- Stars delta per repo
- GitHub traffic / referrer (Insights)
- Clone counts if visible
- Inbound Issues or Discussions mentioning discovery channel

Non-goals for this phase: maximizing social likes without GitHub follow-through.
